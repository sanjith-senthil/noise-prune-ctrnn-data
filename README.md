# Mod-Cog CTRNN Pruning Data Release

This data release contains the trained model checkpoints and frozen result
files used for the paper's quantitative claims. It is intended to pair with the
separate code release for reproducing the H=512 Mod-Cog CTRNN pruning analyses.

## Scope

The deposited model data are the 24 selected H=512 CTRNN checkpoints:

- 8 Mod-Cog tasks
- 3 independently trained model seeds per task
- selected final checkpoint from the 12,000-step two-phase training run
- checkpoint selection by validation sequence accuracy, with validation
  cross-entropy loss as the tie-break

The deposited result files cover:

- main task-retention benchmark, backing Figure 1 and in-text retention values;
- amplification-factor distributions at 80% pruning, backing Figure 2;
- L-NP/S-NP capped-rescale q10-to-q100 sweep, backing Figure 3;
- training-outcome summaries, backing Table 1;
- statistical outputs for the main 84-comparison task-retention family and the
  18-comparison capped-rescale family.

The Mod-Cog task generator itself is not deposited here. Calibration, scoring,
evaluation, and validation batches are regenerated by the code release from the
recorded config seeds rather than stored as raw batch tensors.

## Layout

- `checkpoints/`: 24 selected final model checkpoints plus original per-seed
  checkpoint manifests.
- `configs/`: training and analysis configs recording seeds and suite settings.
- `metadata/checkpoint_inventory.csv`: checkpoint paths, SHA-256 hashes, tensor
  keys/shapes, task, model seed, and selection rule.
- `metadata/source_file_map.csv`: mapping from source artifacts to release
  paths, including derived release tables.
- `results/task_retention/`: main seven-method task-preservation raw and
  summary tables, including spectral-abscissa summaries derived from the raw
  `post_rec_ct_abscissa` field.
- `results/amplification_factors/`: candidate-edge `1 / p_ij` distribution
  histograms and run-level summaries at 80% pruning. The release stores binned
  per-edge counts plus run summaries, not one row per candidate edge.
- `results/capped_rescale/`: q10-to-q100 capped-rescale raw rows and plot-level
  summaries with s.e.m. columns, including q-by-sparsity spectral-abscissa
  summaries.
- `results/training_outcomes/`: final selected-checkpoint outcomes by network,
  per-task summaries, and source training result CSVs.
- `results/statistics/`: Wilcoxon signed-rank and exact sign-test outputs,
  Holm-adjusted p-values, and descriptive statistics.

## Reproducibility Notes

Official task-preservation analyses use fixed held-out evaluation batches and
independent fixed pruning-score batches. The relevant seeds are recorded in the
analysis configs:

- score/calibration batch seeds: `100000`, `100001`, `100002`
- held-out evaluation batch seed base: `200000`
- evaluation seed: `0`
- training checkpoint validation batches: 64 batches during training
- task-preservation evaluation batches: 128 fixed held-out batches

Checkpoint tensors use the code release's `CTRNN` state-dict naming:

- `input_layer.weight`, `input_layer.bias`
- `hidden_layer.weight`, `hidden_layer.bias`
- `readout_layer.weight`, `readout_layer.bias`

These correspond to the input, recurrent, and readout weights/biases described
in the paper.

Spectral-abscissa summary tables report both the signed
`post_rec_ct_abscissa` value and `abs(post_rec_ct_abscissa)`. Stochastic
pruning seeds are averaged within each trained network first, then means,
s.d., and s.e.m. are computed across the 24 trained networks. Raw run means are
also included for auditability.

Some released rescale configuration rows contain a matched-diagonal flag. This
flag is nullified by the no-self-connection constraint and is inactive in all
reported runs. Some legacy source labels use `V-NP` for linearized noise-prune;
these labels are identical to `L-NP` as defined in the manuscript.

## Checksums

`manifest.json` and `MANIFEST.sha256` contain SHA-256 checksums for every file
in the release staging directory.
